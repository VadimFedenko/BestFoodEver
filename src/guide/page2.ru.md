# Управляйте Алгоритмом

**1. Слайдеры (0-10):** Устанавливают приоритеты.
*Дешевое = 10* → Приложение высоко оценивает **дешевые** блюда.
*Дешевое = 0* → Выключает приоритет.

**2. Тап по приоритету:** Инвертирует его.
*Дешевое* → *Дорогое*.
Тогда, чем блюдо **дороже**, тем **выше** его оценка.

<img src="./tap_to_invert.webp" alt="Анимация: Нажатие на критерий для инверсии" class="onboarding-animation" />

**3. Экономические зоны:** Меняйте регион → расчет стоимости произойдет по местным ценам.

**4. Пресеты:** Кнопка *Сохранить* сохраняет ваш топ. Пресеты хранятся локально на вашем устройстве.

<details>
<summary><strong>Цены в $/1000ккал?</strong></summary>

Приложение предлагает 3 метрики цены: `$/кг`, `$/порция` и `$/1000ккал`. Зачем нужны все 3?

Сравнивать цену двух блюд за порцию - сложно (порции блюд различаются по размеру)

Сравнивать за кг тоже не всегда практично. Представьте: литр чая стоит копейки, причем будет вкусным и быстрым. В пересчете на кг, чай - лучшее "блюдо". Единственный минус чая - в нем около 0 калорий. 

Поэтому цены зачастую выгодно смотреть в пересчете на калории.
Для этого в приложении 3 различные метрики (а чая тут вообще нет!).

</details>
